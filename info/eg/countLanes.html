Count Lanes (and scatter by them)

This task counts how many sequencing lanes are present and checks which lane each FASTQ sample comes from, so that downstream tasks like alignment and variant calling can be scattered (i.e. run in parallel).

  • A common convention is for the lane number to be indicated in the FASTQ filename or header - e.g. "L001", "L002". This can be extracted and used to separate the data by lane.

    • Illumina flow cells are divided into multiple lanes, which are physically separate channels where the DNA sequencing reactions occur. The number and type of lanes depend on the flow cell and platform used. At Exeter Genomics Laboratory, we use 25B flow cells which have 8 lanes. This supports throughput of 64 genomes at 30x coverage, 750 exomes at 100x coverage or 1000 transcriptomes.

    Because the data from different lanes are independent, we can process each lane separately and in parallel. This scattering approach distributes computational load across multiple nodes, which optimises resource usage and reduces runtime.
    Also, if a job fails for one lane, we only would need to rerun this rather than the entire batch, saving time and resources.

    Later, when we convert FASTQ files to BAM format, the resulting BAMs are sorted and grouped by lane, which maintains this structure for downstream processing.
