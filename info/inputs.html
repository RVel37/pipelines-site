Inputs

The inputs for this pipeline are as follows:

  • Run project ID
  • Run directory
    • All projects (samples or families) are assigned a run project ID and their own directory. This is where all outputs for that particular sample are going to end up. This can be looked up in DNAnexus after within the right main directory (genomes/exomes/targeted > data > families/projects)
  • Docker images; one for MultiQC, and one for run metrics.
    • Docker is a tool that helps you package applications and their dependencies into a container. You can think of a container as a small, self-contained environment where your application can run without being affected by or affecting other programs on your computer. This is important because it ensures the pipeline will work the same way no matter where it is run, regardless of the machine or OS.

Optional inputs

  • Mode: live, develop, or validation. Live by default.
    • Develop: Files are moved to the development project on DNAnexus.
    • Validation: Files are moved to the validation project on DNAnexus.
  • Branch: Specifies which branch of the tREX/EG pipeline to clone during MixedPreprocessing. Main by default.
    • For example, if testing specific developments, you may change to another branch.
